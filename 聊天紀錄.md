### 3. 模型預測卡死問題分析 (Out-of-Distribution 問題)
- **問題描述**：使用者回報在樹莓派端執行 `carema.py` 時，即便 `test.py` 有 68% 準確率，攝影機的預測結果仍完全卡死，不會隨手勢變更。
- **原因分析 (AI 理論推導)**：
  1. **RBF Kernel 數學特性**：SVM 的 RBF Kernel 藉由計算空間距離來判斷相似度。當真實攝影機的背景或光線與訓練集差異極大時，兩者在 4096 維空間中的距離會非常遠，導致 Kernel 計算的相似度趨近於 0，模型最終只會無腦輸出一個固定的 Bias (偏差) 類別。
  2. **攤平像素 (Flatten) 失去空間特徵**：直接把圖片拉平使得背景像素佔據極大影響力，缺乏對背景雜訊的抵抗能力。
  3. **資料分佈不一致**：`test.py` 的測試集與訓練集來自相同拍攝環境（同資料分佈），因此準確率正常；但使用者的 Webcam 環境是全新的分佈 (Out-of-Distribution)，因此導致模型崩潰。
- **解決方向**：這正好能作為報告中「為何要替換模型」的最佳理由。AI 建議後續實作具有特徵擷取能力的 **CNN (如 MobileNetV2)** 或另一種傳統模型 **Random Forest** 進行對比。

### 8. 新模型方向分析與選型結論
- **使用者需求**：先分析兩個新模型該選什麼，重點是要考慮 Raspberry Pi 4 端的推論表現、部署負擔與最終成績。
- **分析結論**：AI 綜合現有資料流程、`carema.py` 的多格式推論支援，以及樹莓派 CPU-only 場景後，建議優先選擇以下兩個新模型：
  1. **HOG + SVM**：保留傳統機器學習路線，但以 HOG 特徵取代單純 flatten 灰階像素，可更有效保留手勢輪廓與局部方向特徵，部署輕量、推論快速，最適合作為 Raspberry Pi 4 上的穩定方案。
  2. **MobileNetV2**：作為輕量化 CNN 代表，對真實攝影機背景、光線與姿態變化的泛化能力通常優於傳統模型，也較符合邊緣裝置情境。
- **不優先選擇的方向**：
  - **Random Forest**：若仍使用原始攤平特徵，往往只是更換分類器，未必真正改善真實鏡頭的 domain shift。
  - **較大 CNN / ResNet 類**：對 Raspberry Pi 4 來說，推論延遲、記憶體與安裝成本偏高。
- **定位策略**：
  - `HOG + SVM` 負責「低成本、易部署、CPU 友善」的傳統路線。
  - `MobileNetV2` 負責「較高泛化能力、較佳真實場景辨識率」的深度學習路線。
  - 兩者搭配適合寫成作業報告中的完整對照組。

### 13. 嘗試以更強的傳統模型取代 HOG + SVM
- **使用者需求**：使用者認為 `HOG + SVM` 表現太差，希望至少換成一個比 baseline SVM (`68.28%`) 更好的傳統模型。
- **第一次替代嘗試：PCA + Linear SVM**
  - 新增 `train/train_pca_svm.py`。
  - 維持原本灰階 `64x64` + flatten 特徵，再接 `StandardScaler + PCA + LinearSVC`。
  - 結果：`61.29%`，仍低於 baseline。
- **第二次替代嘗試：PCA + Random Forest**
  - 新增 `train/train_pca_rf.py`。
  - 以 `StandardScaler + PCA + RandomForestClassifier` 作為第二版傳統模型替代。
  - 結果：`52.15%`，比 baseline 明顯更差。
- **第三次替代嘗試：直接 Random Forest (不做 PCA)**
  - 新增 `train/train_rf.py`。
  - 直接在原始 flatten 特徵上訓練 `RandomForestClassifier`。
  - 結果：`56.45%`，仍低於 baseline。
- **結論**：
  - 在目前資料集與前處理條件下，原本的 baseline `RBF SVM` 已經是目前測過最強的傳統模型。
  - `HOG + SVM`、`PCA + Linear SVM`、`PCA + Random Forest`、`Random Forest` 都沒有超過 baseline。
  - 因此若作業要求必須展示兩個新架構，最穩妥做法會是：
    1. 保留 baseline SVM 當傳統方法參考基準。
